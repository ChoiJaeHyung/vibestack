# Codecademy 기능 분석 — VibeUniv 도입 검토 (수정본)

> 분석일: 2026-03-01
> 수정일: 2026-03-02
> 목적: Codecademy의 핵심 기능을 분석하고, VibeUniv에 도입 가능한 항목을 우선순위화

---

## 1. Codecademy 핵심 기능 요약

### 1.1 게이미피케이션

| 기능 | 설명 |
|------|------|
| **XP 포인트** | 레슨/연습/프로젝트 완료 시 XP 획득. 누적 XP로 레벨 산정 |
| **배지** | 4종류: 연습 배지(N개 완료), 코스 배지(레슨 완료), 코스 고유 배지, 프로모션 배지 |
| **주간 스트릭** | 주간 목표(기본 3일/주) 달성 시 스트릭 +1. 매주 월요일 리셋. 손실 회피 심리 활용 |
| **주간 학습 목표** | 첫 가입 시 "주 몇 일, 하루 몇 시간" 설정. 목표 대비 진행률 시각화 |
| **레벨 시스템** | 누적 XP 기반 레벨. 높은 레벨 = 전문성 시그널 |

### 1.2 진행률 표시

| 기능 | 설명 |
|------|------|
| **코스 진행률 바** | 레슨/코스 단위 % 진행 바. 즉각적 진행감 제공 |
| **일별 활동 캘린더** | 주간 캘린더(월~일)에 학습한 날 표시. 스트릭과 연동 |
| **마일스톤** | 코스 중간중간 마일스톤 표시. 큰 목표를 작은 단위로 분할 |

### 1.3 완료 피드백

| 기능 | 설명 |
|------|------|
| **축하 애니메이션** | 레슨 완료 시 confetti + 축하 메시지 |
| **배지 획득 알림** | 새 배지 획득 시 모달 팝업 |
| **레벨업 시각 효과** | XP 누적으로 레벨업 시 애니메이션 |

### 1.4 개인화

| 기능 | 설명 |
|------|------|
| **학습 스케줄 설정** | 주간 학습 일수 + 일일 학습 시간 선택 |
| **추천 시스템** | 완료한 코스 기반 다음 코스 추천 |
| **이어서 학습하기** | 대시보드에서 마지막 위치부터 바로 이어서 시작 |

### 1.5 소셜/공유

| 기능 | 설명 |
|------|------|
| **프로필 페이지** | 배지, 완료 코스, 스트릭, 포인트 표시 |
| **공유 카드** | 완료 인증서/배지를 SNS에 공유 가능 |

### 1.6 리텐션

| 기능 | 설명 |
|------|------|
| **이메일 리마인더** | 학습 목표 미달성 시 이메일 알림 |
| **푸시 알림** | 스트릭 끊기기 전 알림 |
| **3단계 강화 루프** | 미시(레슨 완료) → 중간(모듈/스트릭) → 거시(코스/배지) |

## 3. VibeUniv vs Codecademy: 사용자 동기의 근본적 차이

> 이 섹션은 Codecademy 기능을 무비판적으로 가져오지 않기 위한 필터 역할을 한다.

| 차원 | Codecademy | VibeUniv |
|------|-----------|----------|
| **사용자 목표** | "코딩을 배우고 싶다" (장기 스킬 습득) | "내가 만든 앱을 이해하고 싶다" (프로젝트 기반 단기 목표) |
| **학습 기간** | 수개월~수년 (장기 코스) | 2~4주 (프로젝트 1개 커리큘럼 6~8모듈) |
| **콘텐츠** | 범용 커리큘럼 (모든 사용자 동일) | 개인화 (내 프로젝트 코드가 교재) |
| **리텐션 목표** | 구독 유지 (월과금 모델) | 프로젝트 추가 시 재방문 |
| **게이미피케이션 의미** | 장기 동기 부여 필수 | 단기 모멘텀 유지에 초점 |

**시사점:**
- XP/레벨처럼 "무한 누적"을 전제로 하는 시스템은 VibeUniv의 단기 학습 여정에 맞지 않을 수 있음
- 스트릭도 "주간" 단위보다 "학습 경로 내 연속일"이 더 의미 있음
- 공유 카드는 사용자 기반이 커진 후 + 홍보 전략(Product Hunt 런치)과 맞물려 도입하는 게 타이밍 적절

## 4. 도입 후보 기능 상세

### 4.1 모듈 완료 축하 애니메이션

**Codecademy 방식:** confetti 파티클 + "Congratulations!" 메시지 + 획득 XP 표시

**VibeUniv 적용안:**
- 모듈 완료 시 confetti 애니메이션 (canvas-confetti 라이브러리, ~3KB)
- "축하해요! {모듈명} 완료" 모달
- 퀴즈 점수 표시 (있는 경우)
- "다음 모듈로" CTA 버튼

**변경 범위:**
- `components/features/module-content.tsx` — 완료 핸들러에 축하 UI 추가
- 새 컴포넌트 `components/features/celebration-modal.tsx`
- npm: `canvas-confetti`

**난이도:** 낮음 (프론트엔드만, DB 변경 없음)
**예상 임팩트:** 중 (즉각적 성취감 → 다음 모듈 전환율 상승)

### 4.2 대시보드 "오늘 학습" 넛지 배너

> 4.7 이메일 리마인더의 앱 내 경량 버전. DB 변경 없이 즉시 구현 가능.

**VibeUniv 적용안:**
- 대시보드 상단에 조건부 배너 표시:
  - 오늘 학습 활동이 없으면: "🔥 오늘 아직 학습을 시작하지 않았어요! → 이어서 학습하기"
  - 스트릭(Phase B 도입 후)이 끊기기 직전이면: "스트릭이 끊어지기 전에 오늘 학습하세요!"
- 학습 활동이 있으면 배너 숨김 또는 "오늘도 학습 완료 ✓" 표시
- dismiss 가능 (세션 내 1회만 표시)

**변경 범위:**
- `components/features/dashboard-content.tsx` — 조건부 배너 컴포넌트 추가
- 판별 로직: `learning_progress` 테이블에서 오늘 날짜의 `updated_at` 존재 여부 (기존 데이터 활용, DB 변경 없음)

**난이도:** 낮음 (프론트엔드 + 기존 API 데이터 활용)
**예상 임팩트:** 중 (앱 내 리텐션 넛지. 이메일 리마인더 구축 전까지 가벼운 대안)

### 4.3 학습 스트릭 (일간 기반)

**Codecademy 방식:** 주간 목표(기본 3일) 달성 → 스트릭 +1주. 매주 월요일 리셋.

**VibeUniv 차별화:** Codecademy는 수개월짜리 장기 코스라 "주간" 스트릭이 자연스럽지만, VibeUniv의 학습 경로는 6~8개 모듈, 총 10~20시간으로 2~3주면 완료 가능하다. "12주 연속 스트릭"은 VibeUniv에서 도달 불가능한 목표가 되므로, **일간 연속 학습 스트릭**을 채택한다.

**VibeUniv 적용안:**
- **일간 스트릭:** 연속 학습일 카운트 (예: "5일 연속 학습 중 🔥")
- 불꽃 아이콘 + "N일 연속 학습 중" 대시보드 위젯
- 주간 캘린더 (월~일) — 학습한 날 체크 표시
- "오늘 학습하면 스트릭 유지!" 넛지 (4.2 배너와 연동)

**주간 학습 목표와 통합:**
- 첫 학습 시작 시 또는 설정 페이지에서 목표 설정 모달
- 선택지: 주 2일 / 3일(추천) / 5일 / 매일
- 대시보드에 "이번 주 목표: 2/3일 완료" 진행 바
- 주간 목표 달성 시 보너스 표시 ("이번 주 목표 달성! 🎉")

**변경 범위:**
- DB: `user_streaks` 테이블 신규
  ```sql
  id UUID PK
  user_id UUID FK UNIQUE
  current_streak INT DEFAULT 0       -- 현재 연속 일수
  longest_streak INT DEFAULT 0       -- 최장 기록
  weekly_target INT DEFAULT 3        -- 주간 목표 일수
  last_active_date DATE              -- 마지막 학습일
  week_active_days INT DEFAULT 0     -- 이번 주 학습 일수
  week_start_date DATE               -- 이번 주 시작일 (월요일)
  created_at, updated_at
  ```
- 스트릭 업데이트 로직:
  - 모듈 완료 또는 학습 활동 시 `last_active_date` 업데이트
  - `last_active_date`가 어제이면 `current_streak += 1`
  - `last_active_date`가 어제보다 이전이면 `current_streak = 1` (리셋)
  - `last_active_date`가 오늘이면 변경 없음 (중복 방지)
  - `longest_streak = max(current_streak, longest_streak)` 항상 갱신
- `server/actions/learning.ts` — 모듈 완료 시 스트릭 업데이트 호출
- `components/features/dashboard-content.tsx` — 스트릭 위젯
- `components/features/streak-widget.tsx` — 신규 컴포넌트 (불꽃 아이콘 + 연속일 + 주간 캘린더)
- `app/(dashboard)/settings/page.tsx` — 학습 목표 설정 섹션 추가

**난이도:** 중
**예상 임팩트:** 높음 (리텐션 핵심 드라이버. 학습 경로 완료까지의 모멘텀 유지)

---

### 4.4 일별 활동 캘린더 위젯

**VibeUniv 적용안:**
- 대시보드에 주간 캘린더 (월~일) 표시
- 학습한 날: 보라 채움 원 / 학습 안 한 날: 빈 원 / 오늘: 보라 링 (학습 전) 또는 보라 채움 (학습 후)
- 4.3 스트릭과 동일 데이터 활용 (`user_streaks.week_active_days` + `last_active_date`)

**변경 범위:**
- `components/features/streak-widget.tsx`에 포함 (4.3과 동시 구현)
- 추가 DB 변경 없음 (4.3의 `user_streaks` 활용)

**난이도:** 낮음 (4.3과 함께 구현)
**예상 임팩트:** 중 (시각적 학습 패턴 확인 → 습관 강화)

### 4.5 배지/업적 시스템

**Codecademy 방식:** 연습 N개 완료 배지, 코스 완료 배지, 특수 배지(이벤트), 프로모션 배지.

**VibeUniv 적용안:**

| 배지 | 조건 | 아이콘 |
|------|------|--------|
| 첫 발자국 | 첫 모듈 완료 | 👶 |
| 꾸준한 학습자 | 7일 연속 스트릭 | 🔥 |
| 퀴즈 마스터 | 퀴즈 3회 연속 만점 | 🧠 |
| 코드 챌린저 | 챌린지 10개 해결 | 💻 |
| 풀스택 탐험가 | 커리큘럼 1개 전체 완료 | 🎓 |
| 다재다능 | 3개 이상 기술 스택 학습 | 🌈 |
| AI 친구 | AI 튜터 대화 50회 | 🤖 |
| 속도광 | 모듈 1개를 10분 내 완료 | ⚡ |

**변경 범위:**
- DB: `badges` (정의), `user_badges` (획득 기록) 테이블 신규
  ```sql
  -- badges
  id UUID PK
  slug VARCHAR UNIQUE            -- 'first_step', 'quiz_master' 등
  name VARCHAR                   -- '첫 발자국'
  description TEXT
  icon VARCHAR                   -- 이모지 또는 아이콘명
  condition_type VARCHAR          -- 'module_complete', 'streak', 'quiz_score' 등
  condition_value INT             -- 조건 수치
  created_at

  -- user_badges
  id UUID PK
  user_id UUID FK
  badge_id UUID FK
  earned_at TIMESTAMP
  UNIQUE(user_id, badge_id)
  ```
- `server/actions/badges.ts` — 배지 조건 체커 (모듈 완료/스트릭 갱신 시 호출)
- `components/features/badge-earned-modal.tsx` — 획득 시 축하 모달
- 대시보드 + 프로필에 배지 그리드 표시

**난이도:** 중
**예상 임팩트:** 중~높음 (장기 목표 부여, 수집 욕구 자극)

### 4.6 학습 경로 공유 카드

**Codecademy 방식:** 완료 인증서를 이미지로 생성하여 SNS 공유 가능.

**VibeUniv 적용안:**
- 커리큘럼 완료 시 동적 OG 이미지 생성 (기존 `opengraph-image.tsx` 패턴 활용)
- 공유 카드 내용: 프로젝트명, 학습 기술, 진행률, 배지, 소요 시간
- 공유 버튼: 트위터, 링크 복사

**변경 범위:**
- `app/api/share/learning-card/route.tsx` — 동적 이미지 생성 API
- 학습 경로 페이지에 공유 버튼 추가

**난이도:** 낮음 (기존 OG 이미지 생성 패턴 재활용)

**타이밍 참고:** 공유 카드는 사용자가 커리큘럼을 완료해야 작동하므로, 초기 사용자가 적은 단계에서는 효과가 제한적이다. **Product Hunt 런치 직전에 도입**하면 런치 시 공유 효과를 극대화할 수 있다.

**예상 임팩트:** 중 (바이럴 + 신규 유저 유입. 단, 사용자 기반이 커질수록 효과 증가)

### 4.7 학습 일정 리마인더

**Codecademy 방식:** 목표 미달성 시 이메일 + 푸시 알림으로 복귀 유도.

**VibeUniv 적용안:**

**Phase 1 — 앱 내 넛지 (4.2에서 이미 구현):**
- 대시보드 배너로 "오늘 학습 안 했어요" 알림. DB/인프라 변경 없이 즉시 가능.

**Phase 2 — 이메일 리마인더 (Phase D에서 구현):**
- 주간 목표 미달성 시 이메일 알림 ("이번 주 목표까지 1일 남았어요!")
- 스트릭 끊기기 전 알림 ("스트릭이 끊어지기 전에 오늘 학습하세요!")
- Supabase Edge Function + cron (주 2회)
- 이메일 템플릿 (Resend 또는 Supabase 내장)
- 알림 설정 토글 (설정 페이지)

**난이도:** Phase 1 낮음 / Phase 2 중~높음 (서버리스 함수 + 이메일 인프라 필요)
**예상 임팩트:** 높음 (이탈 사용자 복귀에 가장 효과적)

---

## 5. XP/레벨 시스템 — 보류 (사유)

**Codecademy 방식:** 모든 학습 행동에 XP 부여. 누적 XP로 레벨 결정.

**보류 사유:**

VibeUniv 사용자는 프로젝트 1~2개를 분석하고 핵심을 이해하면 목적을 달성한다. Codecademy처럼 수개월~수년에 걸친 장기 여정이 아니기 때문에, XP/레벨의 "무한 누적" 전제가 VibeUniv의 단기 학습 여정과 맞지 않는다.

구체적으로:
- Free 사용자가 프로젝트 3개를 분석하고 떠나는 시나리오에서 "Lv.3"이 의미 있는 동기부여가 되지 않음
- XP 테이블, 레벨 계산 로직, UI 등 구현 비용 대비 효과가 불확실
- 배지 시스템(4.5)이 이미 행동별 보상감을 제공

**재검토 조건:**
- 월간 활성 사용자 500명 이상 달성 후 리텐션 데이터 분석
- Pro 사용자의 평균 학습 기간이 4주 이상으로 확인될 경우
- 배지 시스템 도입 후 사용자 피드백에서 "더 지속적인 보상"에 대한 수요가 확인될 경우

이 시점에서 XP/레벨 도입 여부를 데이터 기반으로 재판단한다.

**보류 중인 설계안 (참고용):**

| 행동 | XP |
|------|-----|
| 모듈 완료 | +100 |
| 퀴즈 만점 | +50 |
| 챌린지 해결 | +30 |
| 일일 학습 (첫 활동) | +20 |
| 스트릭 유지 (주간) | +50 |
| 커리큘럼 전체 완료 | +500 |

레벨: 0~100XP = Lv.1, 100~300 = Lv.2, 300~600 = Lv.3, ...

## 6. 구현 우선순위 로드맵

### Phase A: Quick Wins (1~2일)

즉각적 체감 개선. DB 변경 없음.

| # | 기능 | 난이도 | DB 변경 | 비고 |
|---|------|--------|---------|------|
| A1 | 모듈 완료 축하 애니메이션 | 낮음 | 없음 | `canvas-confetti` + 모달 |
| A2 | 대시보드 "오늘 학습" 넛지 배너 | 낮음 | 없음 | 기존 데이터로 판별 |

### Phase B: 핵심 리텐션 (3~5일)

학습 습관 형성의 핵심 루프.

| # | 기능 | 난이도 | DB 변경 | 비고 |
|---|------|--------|---------|------|
| B1 | 학습 스트릭 (일간) + 주간 목표 | 중 | `user_streaks` 테이블 | 일간 연속일 기반 |
| B2 | 일별 활동 캘린더 위젯 | 낮음 | B1과 동일 | B1과 동시 구현 |

### Phase C: 업적 시스템 (3~5일)

장기 목표 부여 + 수집 동기.

| # | 기능 | 난이도 | DB 변경 | 비고 |
|---|------|--------|---------|------|
| C1 | 배지/업적 시스템 | 중 | `badges`, `user_badges` 테이블 | 8종 배지 |

### Phase D: 바이럴 + 리텐션 (5~7일)

사용자 기반 성장 단계에서 도입. Product Hunt 런치 전 완료 목표.

| # | 기능 | 난이도 | DB 변경 | 비고 |
|---|------|--------|---------|------|
| D1 | 학습 경로 공유 카드 | 낮음 | 없음 | 런치 직전 도입 |
| D2 | 이메일 리마인더 | 중~높음 | Edge Function + 이메일 | 이탈 복귀 |

### 보류

| # | 기능 | 사유 | 재검토 조건 |
|---|------|------|------------|
| — | XP/레벨 시스템 | 단기 학습 여정에 무한 누적 보상 부적합 | MAU 500+ 달성 후 리텐션 데이터로 판단 |

## 7. 도입하지 않는 기능 (사유)

| 기능 | 사유 |
|------|------|
| 커리어 패스 | VibeUniv는 "내 프로젝트 기반" 학습이라 범용 커리어 경로와 성격이 다름 |
| 리더보드 | 개인 학습 플랫폼. 경쟁보다 자기 성장에 집중 |
| 인터랙티브 코드 실행 | 서버 비용 큼. 현재 코드 챌린지(에디터+정답 토글)로 충분 |
| 포럼/커뮤니티 | 초기 단계에서 커뮤니티 운영 부담 큼. AI 튜터가 대안 |
| 인증서 발급 | 프로젝트별 커리큘럼이라 범용 인증서 의미 약함 |
